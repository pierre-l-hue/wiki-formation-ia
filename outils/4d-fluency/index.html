    4D Fluency¶

    Les 4 compétences de l’expert IA.

    🎯 À retenir¶

    Le framework 4D Fluency aide à mieux travailler avec l’IA en combinant : - le bon niveau d’usage ; - une bonne formulation de la demande ; - une évaluation rigoureuse des résultats ; - une posture responsable dans la diffusion.

    3 façons de travailler avec l’IA¶

    Mode Description Exemples
    Automation L’IA exécute une tâche à votre place, sur vos instructions précises. Rédiger un email, faire un résumé, générer un tableau
    Augmentation Vous et l’IA co-créez en itérant ensemble, dans un dialogue. Écrire un rapport, brainstormer, affiner une stratégie
    Agency Vous configurez une IA pour qu’elle agisse de façon autonome, selon vos règles. Chatbot client, assistant tuteur, agent d’onboarding

    À noter

    L’IA n’est pas un outil monolithique. Selon la tâche, le niveau d’autonomie souhaité et le contexte, on interagit différemment avec elle. On passe souvent de l’un à l’autre dans un même projet.

    Le framework des 4D¶

    Deux boucles complémentaires structurent ce cadre : - une boucle stratégique pour les grandes décisions ; - une boucle tactique pour le travail du quotidien.

    Délégation

    Choisir comment utiliser l’IA : - définir ce qu’on veut accomplir ; - répartir les rôles entre humain, IA et collaboration ; - choisir le bon outil selon le besoin, le budget, la confidentialité ou le cadre RGPD.

    Description

    Communiquer avec l’IA : - préciser le résultat attendu ; - décrire éventuellement le processus ; - cadrer aussi le comportement attendu de l’outil.

    Discernement

    Évaluer les résultats : - vérifier si la réponse est exacte, utile et exploitable ; - repérer hallucinations, approximations et biais ; - juger aussi la qualité de la collaboration avec l’IA.

    Diligence

    Être un utilisateur responsable : - identifier les risques et impacts ; - rendre l’usage de l’IA explicite quand c’est nécessaire ; - vérifier, valider et assumer ce qui est diffusé.

    Description : 3 niveaux progressifs¶

    Niveau 1 — Résultat

    Décrire directement ce que l’on veut obtenir. **Exemple** `Rédige une fiche produit de 150 mots, ton professionnel, pour une PME BtoB.`

    Niveau 2 — Processus

    Décrire les étapes de travail attendues. **Exemple** `Commençons par le plan, puis on développe section par section.`

    Niveau 3 — Comportement

    Définir la posture et les règles d’action de l’IA. **Exemple** `Tu es un assistant commercial. Tu réponds toujours de façon concise et tu poses une question de qualification.`

    Comment ça s’enchaîne dans un projet réel¶

    1. Délégation : je définis mon objectif et je choisis mon outil.
    2. Description : je formule ma demande précisément.
    3. Discernement : j’évalue le résultat obtenu.
    4. Diligence : je valide, j’assume et je diffuse de façon transparente.

    Lecture en boucle¶

    • Boucle tactique : Description ↔ Discernement.
      Si le résultat ne correspond pas, on reformule, on ajuste et on réessaie.

    • Boucle stratégique : Diligence ↔ Délégation.
      En validant, on découvre parfois que le choix initial d’outil ou de mode n’était pas optimal.

    Bonnes pratiques¶

    Repères utiles¶

    • Toujours définir l’objectif avant de choisir l’outil.
    • Adapter son mode d’usage (Automation, Augmentation, Agency) selon la tâche.
    • Un bon prompt combine résultat attendu, contexte et format.
    • Ne jamais diffuser un résultat IA sans l’avoir relu, vérifié et validé.
    • Être transparent sur l’usage de l’IA quand le contexte l’exige.
