*Usages, souveraineté et limites*

!!! info "Évolution récente"
    En mai-juin 2026, « Le Chat » est devenu **Vibe** (même compte, même historique). Le repositionnement : passer d'un chatbot conversationnel à un agent unifié pour le travail et le code, avec 3 logiques d'usage : **Chat** (échange direct), **Work** (tâches complexes multi-étapes), **Code** (développement).

## Comment utiliser Mistral au quotidien

- **Chercher et résumer** — poser une question, lancer une recherche, synthétiser un sujet
- **Rédiger et structurer** — un mail, un plan, une note, une première version de document
- **Traiter une tâche complexe** — mode Work pour enchaîner plusieurs étapes
- **Coder ou corriger du code** — mode Code, extension VS Code, terminal
- **Exploiter des documents** — extraire du texte structuré à partir de PDF, formulaires, tableaux (OCR)

## Chat, Work, Code : quel mode choisir ?

| Mode | Pour |
|---|---|
| **Chat** | Poser une question, reformuler, résumer, réponse rapide |
| **Work** | Demande plus longue, plusieurs étapes, contraintes ou sources à exploiter |
| **Code** | Générer, corriger, tester, refactoriser du code |

!!! tip "Astuce"
    Pour un besoin simple, commencer en Chat ; dès qu'il y a plusieurs étapes ou un livrable structuré, basculer vers Work.

## Les plus de Mistral

- Alternative européenne aux grands acteurs américains
- Plus de choix pour l'hébergement (cloud ou serveurs choisis par l'organisation)
- Utile pour les données sensibles
- Pas limité au chat — documents, code, tâches avancées

## Les limites de Mistral

- Pas toujours le plus simple pour débuter
- Un outil qui demande des vérifications comme tout autre
- Hébergement, accès aux données, dépendance à un fournisseur : à vérifier au cas par cas
- Moins connu du grand public

## Souveraineté IA : 4 niveaux à distinguer

1. **Entreprise** — l'éditeur est-il français ou européen, sous quelles règles ?
2. **Infrastructure** — où tourne l'outil, dans quel cloud ou serveurs ?
3. **Données** — qui héberge, qui gère, qui peut y accéder ?
4. **Technologie** — l'outil peut-il être déplacé sans dépendre totalement d'un seul fournisseur ?

!!! note "À noter"
    Européen ne veut pas toujours dire souverain de bout en bout : il faut vérifier l'entreprise, l'hébergement, les données et la technologie.

## Mistral en équipe ou en organisation

- Se connecter à une messagerie, un espace de documents, un outil métier
- Automatiser des échanges via API
- Créer des assistants spécialisés (veille, brief...)
- Définir des règles claires d'accès, d'usage et de contrôle

## Quand choisir Mistral ?

- Quand on cherche une IA européenne
- Quand on veut aller plus loin que le simple chat (documents, code)
- Quand on travaille en équipe et veut encadrer les usages
- Quand le besoin réel le justifie — pas seulement parce qu'il est européen

## Bonnes pratiques

- Partir du besoin réel
- Choisir le bon mode d'usage
- Vérifier les règles autour des données
- Garder une validation humaine sur les contenus importants
