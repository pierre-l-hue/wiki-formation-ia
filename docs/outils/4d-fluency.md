# 4D Fluency

*Les 4 compétences de l'expert IA*

## 3 façons de travailler avec l'IA

| Mode | Description | Exemples |
|---|---|---|
| ⚙️ **Automation** | L'IA exécute une tâche à votre place, sur vos instructions précises | Rédiger un email, faire un résumé, générer un tableau |
| 🤝 **Augmentation** | Vous et l'IA co-créez en itérant ensemble — dialogue | Écrire un rapport, brainstormer, affiner une stratégie |
| 🤖 **Agency** | Vous configurez une IA pour qu'elle agisse de façon autonome, selon vos règles | Chatbot client, assistant tuteur, agent d'onboarding |

!!! note "À noter"
    L'IA n'est pas un outil monolithique. Selon la tâche, le niveau d'autonomie souhaité et le contexte, on interagit différemment avec elle — et on passe souvent de l'un à l'autre au sein d'un même projet.

## Le framework des 4D

Deux boucles complémentaires : une **boucle stratégique** (les grandes décisions) et une **boucle tactique** (le travail au quotidien).

### Délégation — choisir comment utiliser l'IA

- Définir précisément ce qu'on veut accomplir, décomposer la tâche en parties humaines/IA/collaboratives
- Comprendre les forces et limites de chaque outil IA, choisir l'outil adapté (budget, RGPD, confidentialité)
- Savoir quelles parties confier à l'IA et lesquelles garder pour l'humain : jugement, éthique, relation

### Description — communiquer avec l'IA

Trois niveaux progressifs :

1. **Niveau 1 — Description du résultat** : « Rédige une fiche produit de 150 mots, ton professionnel, pour une PME BtoB »
2. **Niveau 2 — Description du processus** : « Commençons par le plan, puis on développe section par section »
3. **Niveau 3 — Description du comportement** : « Tu es un assistant commercial. Tu réponds toujours de façon concise et tu poses une question de qualification »

### Discernement — évaluer les résultats

- **Évaluer le résultat** — exact, pertinent, utilisable ? Repérer hallucinations, approximations, biais
- **Évaluer le processus collaboratif** — la façon de travailler avec l'IA était-elle efficace ?
- **Évaluer le comportement IA** — si l'IA agit en autonomie, l'expérience est-elle bonne ?

### Diligence — être un utilisateur IA responsable

- Identifier biais, risques et impacts de ce qu'on produit avec l'IA
- Indiquer clairement quand et comment l'IA a été utilisée
- Vérifier, valider et assumer ce qu'on diffuse — même si l'IA l'a produit

## Comment ça s'enchaîne dans un projet réel

1. **Délégation** — je définis mon objectif et je choisis mon outil
2. **Description** — je formule ma demande précisément
3. **Discernement** — j'évalue le résultat obtenu
4. **Diligence** — je valide, j'assume, je diffuse de façon transparente

- **Boucle tactique** : Description ↔ Discernement (le résultat ne correspond pas ? on reformule, on ajuste, on réessaie)
- **Boucle stratégique** : Diligence ↔ Délégation (en validant, on découvre parfois que le choix initial d'outil n'était pas optimal)

!!! success "Bonnes pratiques"
    Toujours définir l'objectif avant de choisir l'outil · Adapter son mode (Automation/Augmentation/Agency) selon la tâche · Un bon prompt = résultat attendu clair + contexte + format · Ne jamais diffuser un résultat IA sans l'avoir relu, vérifié et validé · Être transparent sur l'usage de l'IA
