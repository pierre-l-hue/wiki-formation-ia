# Cas pratique — Ressources humaines

## Situation

Vous travaillez sur des documents RH : annonces, notes internes, grilles d’entretien, comptes-rendus ou communications aux salariés.

Votre objectif : produire des contenus clairs, inclusifs et conformes, sans exposer de données personnelles.

---

## Tâches où l’IA peut aider

| Tâche | Aide possible de l’IA | Vigilance |
|---|---|---|
| Rédiger une offre d’emploi | Clarifier les missions et le profil | Éviter les biais discriminatoires |
| Préparer une grille d’entretien | Structurer les questions | Garder une validation humaine |
| Reformuler une note interne | Simplifier le langage | Rester précis juridiquement |
| Analyser un texte RH | Repérer les biais ou formulations excluantes | Ne pas transmettre de données personnelles |

---

## Prompt utile

    Tu es expert RH et communication inclusive.

    Objectif :
    Analyse cette offre d’emploi et propose une version plus claire, inclusive et non discriminatoire.

    Contraintes :
    - ne pas modifier le niveau réel du poste ;
    - éviter les stéréotypes ;
    - utiliser un langage simple ;
    - signaler les formulations à risque ;
    - proposer une version améliorée.

    Texte :
    [coller ici une version anonymisée de l’offre]

---

## À éviter

!!! warning "À éviter"
    Ne collez pas dans une IA des informations RH nominatives :
    
    - nom d’un salarié ;
    - salaire ;
    - arrêt maladie ;
    - évaluation annuelle ;
    - conflit interne ;
    - sanction disciplinaire.

---

## Bon réflexe

!!! success "Bon réflexe"
    Pour les sujets RH sensibles, utilisez l’IA comme aide à la rédaction ou à la relecture.
    
    Ne laissez jamais l’IA décider seule d’un recrutement, d’une sanction ou d’une évaluation.
